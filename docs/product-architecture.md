# 产品架构｜Paopao AI 数字分身

## 1. 产品一句话

泡泡是一个面向个人长期成长的 AI 数字分身，它通过飞书/Coze 接收用户的多模态生活输入，沉淀长期记忆，构建自我模型，并主动推动用户完成认知、财富、领导力、表达和世界经验的复利增长。

## 2. 用户画像

目标用户不是只想“记录心情”的人，而是：

- 有强烈成长欲望
- 有现实野心和财富目标
- 每天产生大量想法和信息输入
- 希望 AI 长期理解自己
- 希望有人持续提醒自己回到主线
- 愿意把生活、阅读、旅行、计划、社交和欲望都纳入个人成长系统

## 3. 需求层级

### L1 记录

用户要快速捕捉碎片，不希望丢失想法。

### L2 整理

用户希望 AI 自动提炼重点，归档到目标、人物、地点、阅读、日程等系统。

### L3 理解

用户希望 AI 逐渐知道“我是谁、我要什么、我的长期主线是什么”。

### L4 主动

用户希望 AI 在早晨、晚上、目标节点、偏离计划时主动出现。

### L5 数字分身

用户希望 AI 能成为更高维、更清醒、更正向、更高效版本的自己。

## 4. 核心模块

### Capture Layer

接收输入：

- 文本
- 语音
- 图片
- 链接
- 文件
- 日程
- 地点
- 天气

### Memory Layer

四类记忆：

- Raw Archive：原始输入
- Structured Memory：人物、地点、目标、书籍、机会
- Vector Memory：RAG 检索
- Self Model：用户长期画像

### Reasoning Layer

负责判断：

- 这条输入是什么类型？
- 是否关联长期目标？
- 是否需要更新用户画像？
- 是否需要主动提醒？
- 是否需要生成日记/计划/复盘？

### Interaction Layer

输出形式：

- 飞书回复
- 主动提醒
- 每日总结
- 周复盘
- Web 书架
- 社交媒体卡片

## 6. 关键交互场景

### 场景 1：用户发欲望

用户：

> 我想赚很多钱，但不知道从哪里开始。

泡泡：

> 我收到了：这是财富目标，不是空泛情绪。  
> 下一步我们先拆三个问题：你能提供什么价值？服务谁？30 天内如何验证最小现金流？

### 场景 2：用户发链接

泡泡要：

1. 保存原链接
2. 抽取核心内容
3. 记录用户感想
4. 判断是否关联阅读、商业、社交、人性、技术
5. 未来 RAG 可检索

### 场景 3：早晨主动提醒

> 早上好。今天不是普通一天，是你七年跃迁系统里的一个节点。  
> 先做一件能产生复利的事：读一点硬书、推进一个目标、联系一个关键人，或把一个欲望写成计划。

## 7. 成功指标

### 用户行为指标

- 日均输入次数
- 多模态输入占比
- 主动提醒后的任务回流率
- 周复盘打开率
- 目标更新频次

### AI 体验指标

- 回复是否关联过去记忆
- 用户是否觉得“越来越懂我”
- 主动提醒是否打扰
- 语气是否稳定
- 是否给出可执行下一步

### 长期价值指标

- 用户目标完成率
- 阅读/行动/复盘频次提升
- 关键项目推进速度
- 用户自我理解清晰度提升
